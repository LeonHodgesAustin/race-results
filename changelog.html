Changelog

All notable changes to racetelemetry are recorded here, newest first. This file is written for anyone browsing a published dashboard — not just developers — so entries describe what changed for you as a viewer or driver, in plain language.

[Unreleased]

[1.1.0] - 2026-07-20

Added

  • A "Map-matched" map trace option, alongside Raw/Smoothed/Corrected — snaps a lap's position to the saved reference line using how far the car has travelled (not its raw, possibly-biased GPS position) to figure out where along the track it actually is. For a session with severe, sustained GPS bias — smoothly wrong for tens of seconds at a time, not just brief jitter — this is the only trace that stays usable, at the cost of throwing away any real within-lap line variation for the stretches it engages on. Meant for comparing lap *progress* (where was I faster or slower) on an otherwise-untrustworthy session, not for line analysis. Disabled until a reference line has been built for the track (`racetelemetry trackline`).

Fixed

  • A single bad GPS reading can no longer teleport a lap's map trace. Occasionally a GPS fix reports itself as perfectly good (fine signal quality) while its position is simply wrong by hundreds of metres — usually right after a brief signal interruption. That fix is now recognised as impossible (no racecar covers that distance that fast) and ignored, the same way a genuinely missing fix already was, instead of being trusted and dragging the smoothed trace, the reference line, and the "off-line" check along with it.

[1.0.0] - 2026-07-20

Added

  • The dashboard — one page per session with a map, a lap-by-lap stats table, speed and "delta vs. reference lap" charts, and sector/corner splits, replacing a pile of separate map/chart files you used to have to flip between.
  • Track dashboards — a second dashboard that combines every session ever recorded at a given track into one page, so you can compare a lap from today against a lap from months ago, not just laps within one outing.
  • Lap replay — play back one or several laps at once on the map with a scrub bar and 1x/2x/4x speed control: a ghost marker per lap moves along the track in sync, with a live table of each lap's speed and its gap to the reference lap at that same moment.
  • A published, shareable site (`racetelemetry publish`) — builds a folder of dashboards you can put on GitHub Pages so a co-driver, parent, or teammate can browse results themselves, without you emailing files around.
  • Automatic track recognition — the recorder figures out which track you're at from the first GPS fix of the session; no need to tell it by hand.
  • GPS quality tools — a smoothing filter that cuts GPS jitter, a "reference line" built from the median of every lap ever recorded at a track, a tool to flag laps that drifted off that line (or off the track entirely), and a slow-drift correction tool for GPS that wanders over the course of a session.
  • G-force colouring on the map — colour a lap's trace by how hard the car is working (combined lateral + braking/accel G), the same colour scale as the standalone `gg-map` view.
  • In-dashboard gate editing — drag sector and corner markers directly on the map to fine-tune where split times are measured, with a one-click export you (or the owner) can paste into the track's configuration to make it permanent for everyone.
  • A km/h ↔ mph toggle for anyone more comfortable reading speeds in imperial units.
  • A mobile-friendly layout — the lap list and replay controls collapse down to a single line on a phone screen instead of crowding it.

Changed

  • Lap timing accuracy — a lap's start/finish crossing is now stamped at the midpoint of the car's pass through the finish area rather than the moment it exits, which used to drift with how fast the car was going. Where a precise finish *line* (rather than just a circle) has been set up for a track, timing is tighter still — accurate to roughly ±10-20 milliseconds instead of ±100ms+. Sessions recorded before this change can be re-timed after the fact without re-recording.
  • The dashboard's delta chart (how far ahead/behind you are through the lap) now compares against whichever lap is fastest among the ones you've currently ticked on, rather than always the single fastest lap of the whole session.

Fixed

  • A single bad GPS reading could previously stretch a lap's whole distance scale, leaving sector and corner markers stranded well past where that lap's own speed trace actually ends on the chart. Distance calculations now discard those physically-impossible jumps.
  • A session with drift-corrected GPS data used to permanently greyed-out the "Smoothed" trace option on the map, even though it had already been computed. Both the smoothed and corrected traces are now kept and offered as separate choices.
